# 2026-06-20（六）模拟面试闭环 D2：评分 + 反馈 + 路由

## 目标

闭环的"环"成立：作答被知识库 grounded 地评分，弱点反馈决定下一题方向。

## 任务

- [ ] **评分节点**：输入（题目、参考要点、用户作答、检索到的相关 kb 块），
  LLM 输出 JSON：`score(1-5), strengths[], gaps[], evidence_ids[]`。
  评分 prompt 强调"以证据为准，不臆测"，gaps 必须指向具体知识点。
- [ ] **反馈节点**：把评分转成对用户的简短反馈（做对了什么、缺什么、建议看哪条引用）。
- [ ] **弱点路由**：score < 3 的 topic 进 `weak_areas`；下一题优先从 weak_areas
  相邻知识点出（同主题换角度，不要原题重复）；连续 2 题同主题低分则换主题。
- [ ] **收尾节点**：`round_count >= max_rounds`（默认 5）时生成总结：
  各题得分表、薄弱方向、针对性复习清单（绑 source_ids）。
- [ ] 串成完整子图，CLI 全流程跑通一轮 5 题模拟面试。
- [ ] 单测：评分 JSON 校验、弱点路由逻辑（纯函数）、收尾总结结构，3-4 个。

## 验收

- CLI 完整跑完 5 题：出题 → 作答 → 评分反馈 → 弱点影响下一题 → 总结。
- 故意答错某主题，能观察到后续题目向该主题倾斜。
- `uv run pytest` 全绿。

## 备注

- 评分是 LLM-as-judge，README 里如实标注其局限（主观性、对长答案的偏好）。
