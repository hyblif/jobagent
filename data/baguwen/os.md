# 操作系统面试八股

## 进程与线程

Q1: 进程和线程的区别是什么？
A: 进程是资源分配的最小单位，拥有独立的地址空间、文件描述符和系统资源；线程是 CPU 调度的最小单位，同进程内的线程共享地址空间和资源。进程切换开销大（需切换页表、刷 TLB），线程切换开销小。一个进程至少包含一个线程，多线程程序中某个线程崩溃可能导致整个进程终止。

Q2: 什么是死锁？死锁的四个必要条件是什么？
A: 死锁是指两个或多个进程互相等待对方释放资源，导致永久阻塞的状态。四个必要条件：(1) 互斥条件——资源同一时刻只能被一个进程持有；(2) 请求与保持——进程持有资源的同时还在等待新资源；(3) 不可剥夺——已分配的资源不能被强制剥夺；(4) 循环等待——进程间存在环形等待链。破坏任意一个条件即可预防死锁。

Q3: 进程间通信（IPC）有哪些方式？
A: 常见 IPC 方式：(1) 管道（Pipe）——单向，父子进程间；(2) 命名管道（FIFO）——支持无亲缘关系进程；(3) 消息队列——内核维护消息链表，支持结构化消息；(4) 共享内存——最快，需配合信号量同步；(5) 信号量——PV 操作同步控制；(6) 信号——异步通知机制；(7) Socket——支持跨主机通信。共享内存速度最快，因为数据不需要在内核和用户态之间复制。

## 内存管理

Q4: 什么是虚拟内存？分页机制如何工作？
A: 虚拟内存让每个进程拥有独立的逻辑地址空间，通过页表映射到物理内存。分页将虚拟地址空间和物理内存划分为固定大小的页（通常 4KB），页表记录虚拟页号到物理页帧的映射。访问虚拟地址时 MMU 查页表完成转换；若物理内存不足，将最近最少使用的页换出到磁盘（swap），按需调入，实现"内存比物理内存大"的效果。多级页表（x86-64 四级）减少页表本身内存占用，TLB 缓存热点映射加速转换。

Q5: 常见页面置换算法有哪些？
A: (1) OPT（最优）——换出未来最长时间不使用的页，理论最优但无法实现；(2) FIFO——换出最早进入内存的页，实现简单但存在 Belady 异常；(3) LRU——换出最近最长时间未访问的页，性能接近 OPT，常用链表或时钟近似实现；(4) Clock（时钟）——环形缓冲区+访问位，近似 LRU，开销小；(5) LFU——换出访问频率最低的页。Linux 内核使用改进的 Clock 算法（active/inactive 双链表）。

## I/O 模型

Q6: Linux I/O 多路复用：select、poll、epoll 的区别？
A: select：监听 fd 集合（最多 1024），每次调用需把 fd_set 从用户态拷贝到内核，内核线性遍历，返回后用户态再次线性遍历找就绪 fd，O(n) 复杂度。poll：同 select 但无 1024 限制，用链表存储；复杂度仍 O(n)。epoll：基于内核事件通知，`epoll_ctl` 注册 fd，`epoll_wait` 直接返回就绪 fd 列表，O(1) 事件检测，无需反复拷贝。epoll 支持 ET（边缘触发）和 LT（水平触发）；ET 只在状态变化时通知，效率更高但需一次性读完数据。高并发场景（Nginx、Redis）首选 epoll。

Q7: 用户态和内核态的切换开销来自哪里？
A: 系统调用、中断、异常均会引起用户态→内核态切换。开销来源：(1) 保存/恢复寄存器上下文（通用寄存器、PC、EFLAGS）；(2) 切换页表（CR3 寄存器）并刷 TLB；(3) 执行内核代码本身；(4) 返回用户态时安全检查（Spectre/Meltdown 补丁加剧）。频繁系统调用是性能瓶颈，优化手段：批量 I/O、mmap（减少拷贝）、io_uring（减少系统调用次数）、用户态网络栈（DPDK）。
