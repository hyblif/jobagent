# 机器学习基础面试八股

## 训练与泛化

Q1: 什么是过拟合和欠拟合？
A: 过拟合是模型在训练集表现很好但泛化差，通常因为模型太复杂、数据少或噪声多；欠拟合是训练集和测试集表现都差，通常因为模型太简单、特征不足或训练不充分。判断要同时看训练误差和验证误差，而不是只看一个指标。

Q2: 如何缓解过拟合？
A: 常见方法包括增加数据、数据增强、降低模型复杂度、正则化、Dropout、早停、交叉验证、集成方法和清洗标签噪声。大模型场景还可通过参数高效微调、冻结部分参数、控制训练步数和使用验证集监控来缓解。

Q3: 训练集、验证集、测试集分别做什么？
A: 训练集用于拟合模型参数，验证集用于选择模型和调超参，测试集用于最终无偏评估。不能反复根据测试集调参，否则测试集会被“用脏”。数据划分要避免泄漏，例如同一用户或同一时间段的数据跨集合出现。

Q4: 什么是数据泄漏？
A: 数据泄漏是训练阶段使用了预测时不可获得的信息，导致离线指标虚高。例如用未来数据构造特征、同一实体同时出现在训练和测试、归一化时用全量数据统计量。排查泄漏要从数据生成时间、特征可用性和划分方式入手。

Q5: 交叉验证有什么作用？
A: 交叉验证把数据分成多折，轮流用一折验证、其余训练，能更稳定评估模型并充分利用小数据集。常见是 k-fold 和 stratified k-fold。时间序列数据不能随机打乱，要用按时间滚动验证。

## 指标与评估

Q6: 准确率、精确率、召回率、F1 有什么区别？
A: 准确率是整体预测正确比例；精确率是预测为正的样本中有多少真为正；召回率是真正例中有多少被找回；F1 是精确率和召回率的调和平均。类别不均衡时准确率可能误导，需要看 precision/recall/F1 或 PR-AUC。

Q7: ROC-AUC 和 PR-AUC 有什么区别？
A: ROC-AUC 看 TPR 和 FPR 的权衡，在类别较均衡时常用；PR-AUC 看 precision 和 recall，更关注正类质量，适合正负极不均衡场景。比如欺诈检测、故障检测中正例很少，PR-AUC 通常比 ROC-AUC 更有解释力。

Q8: 回归任务常见指标有哪些？
A: MAE 是绝对误差平均，鲁棒性较好；MSE/RMSE 对大误差更敏感；MAPE 表示相对误差，但真实值接近 0 时不稳定；R² 衡量解释方差比例。选择指标要和业务损失一致，例如价格预测可能更关心大额误差。

Q9: 什么是混淆矩阵？
A: 混淆矩阵统计 TP、FP、TN、FN，用于分析分类模型错误类型。它能帮助判断模型是漏报多还是误报多，从而选择阈值和优化方向。面试中可以从混淆矩阵推导 precision、recall、specificity 等指标。

Q10: 离线指标和线上效果为什么可能不一致？
A: 原因包括训练数据和线上分布不同、指标不贴合业务、数据泄漏、反馈延迟、样本选择偏差、系统延迟影响体验、线上策略和模型耦合。解决要做 A/B 实验、监控数据漂移、设计业务指标，并保持训练服务一致性。

## 模型与优化

Q11: L1 和 L2 正则有什么区别？
A: L1 正则倾向产生稀疏权重，可做特征选择；L2 正则倾向让权重变小但不为零，提升稳定性。二者都通过惩罚模型复杂度缓解过拟合。线性模型里 L1 对异常特征更敏感，L2 更平滑。

Q12: 梯度下降有哪些变体？
A: Batch Gradient Descent 用全量数据更新，稳定但慢；SGD 每个样本更新，噪声大但可跳出局部；Mini-batch 是深度学习常用折中。优化器如 Momentum、RMSProp、Adam 通过动量和自适应学习率加速收敛。

Q13: 学习率过大或过小会怎样？
A: 学习率过大可能震荡或发散，loss 不降；过小收敛很慢，可能陷入不佳区域。常用策略包括学习率 warmup、cosine decay、step decay、自适应优化器和学习率搜索。调学习率通常是训练稳定性的关键。

Q14: BatchNorm 和 LayerNorm 的区别是什么？
A: BatchNorm 在 batch 维度统计均值方差，依赖 batch size，常用于 CNN；LayerNorm 在单个样本的特征维度归一化，不依赖 batch，常用于 Transformer。小 batch 或序列模型中 LayerNorm 更稳定。

Q15: 为什么 Transformer 用 attention？
A: Attention 能直接建模序列中任意位置的依赖关系，相比 RNN 更易并行，长距离依赖路径更短。Self-Attention 通过 Q/K/V 计算 token 间权重，多头机制让模型关注不同关系。代价是序列长度平方级复杂度。

## 微调与 LLM

Q16: 微调和 RAG 如何取舍？
A: RAG 适合注入外部知识、需要引用、知识频繁更新的场景；微调适合改变模型风格、格式、任务习惯或学习稳定模式。若问题是“不知道事实”，优先 RAG；若问题是“会但输出方式不稳定”，考虑微调。二者也可组合。

Q17: SFT、LoRA、RLHF 分别是什么？
A: SFT 用监督样本训练模型按期望回答；LoRA 是参数高效微调方法，只训练低秩适配矩阵，成本低；RLHF 用人类偏好训练奖励模型，再优化策略使输出更符合偏好。应用项目中常先做 prompt/RAG，再评估是否需要 LoRA/SFT。

Q18: 什么是 embedding 模型的领域适配？
A: 通用 embedding 在专业术语、公司内部缩写、代码片段上可能召回差。领域适配可以用对比学习或难负例训练，让相关 query-doc 更近、不相关更远。没有训练资源时，可以通过 query rewrite、hybrid search 和 reranker 弥补。

Q19: 什么是数据漂移？
A: 数据漂移是线上输入分布随时间变化，导致模型性能下降。例如用户行为、业务规则、季节性和外部环境变化。需要监控特征分布、预测分布和业务指标，必要时重训模型、更新阈值或回滚策略。

Q20: 机器学习项目如何讲清楚业务价值？
A: 先说明业务问题和基线方法，再说明数据、特征、模型、评估指标和上线方式；重点讲相对基线的改进、延迟/成本、失败案例和监控。面试不只听模型名，更看你是否理解问题、指标和工程闭环。
