# Python 语言特性面试八股

## 并发与 GIL

Q1: 什么是 GIL？它对多线程有什么影响？
A: GIL（Global Interpreter Lock）是 CPython 解释器的全局锁，保证同一时刻只有一个线程执行 Python 字节码。影响：CPU 密集型任务多线程无法利用多核（被 GIL 序列化），性能与单线程相当甚至更差；I/O 密集型任务多线程有效，因为线程等待 I/O 时会释放 GIL。绕过 GIL 的方案：使用多进程（multiprocessing）、C 扩展（numpy 等释放 GIL）、或 Python 3.13+ 的 no-GIL 实验性版本。

Q2: 协程（asyncio）和线程的区别？
A: 线程是 OS 调度的执行单元，切换由内核控制，有上下文切换开销，适合 I/O 密集任务；协程是用户态轻量级并发，通过 `await` 显式让出控制权，在单线程内切换，无锁无竞争，切换开销极小。asyncio 用事件循环驱动协程，适合高并发 I/O（网络请求、数据库查询）。协程缺点：CPU 密集任务无法并行；需要代码全链路 async/await（"传染性"）；阻塞调用会冻结事件循环。

## 语言特性

Q3: 装饰器的工作原理是什么？
A: 装饰器是接受函数作为参数、返回新函数的高阶函数，本质是语法糖：`@deco def f(): ...` 等价于 `f = deco(f)`。实现时通常用 `functools.wraps` 保留原函数的 `__name__`、`__doc__` 等元信息。带参数的装饰器需要三层嵌套（外层接收参数，中层接收函数，内层是 wrapper）。类装饰器通过实现 `__call__` 方法实现。常见应用：缓存（`@lru_cache`）、权限校验、计时、重试。

Q4: 生成器和迭代器的区别？yield 的工作机制？
A: 迭代器是实现了 `__iter__` 和 `__next__` 的对象，可以用 `next()` 逐步取值，一次性消耗。生成器是用 `yield` 语句定义的特殊迭代器，函数调用返回生成器对象，每次 `next()` 执行到下一个 `yield` 并暂停，保存完整的栈帧状态（局部变量、执行位置）。优势：惰性求值，按需生成数据，内存占用极小（无需一次性加载所有数据）。`yield from` 委托给子生成器，简化嵌套生成器代码。

Q5: Python 的深拷贝与浅拷贝区别？
A: 浅拷贝（`copy.copy()`）创建新容器对象，但内部元素仍是原始对象的引用；修改可变子对象会影响原始对象。深拷贝（`copy.deepcopy()`）递归复制所有嵌套对象，完全独立。切片 `[:]` 和 `list()` 都是浅拷贝。不可变对象（int、str、tuple 含不可变元素）浅拷贝和深拷贝效果相同。注意：`deepcopy` 处理循环引用会通过备忘录（memo dict）避免无限递归。

## 内存管理

Q6: Python 的垃圾回收机制是什么？
A: CPython 主要使用引用计数：每个对象维护引用计数器，降为 0 时立即释放。为处理循环引用（A→B→A），CPython 额外有分代垃圾回收器（gc 模块），将对象分为 0/1/2 代，定期检测并清理循环引用。引用计数的优点：对象释放及时，无停顿；缺点：维护计数有开销，无法处理循环引用。`__del__` 析构器和 weakref 弱引用是相关机制。PyPy 等实现采用不同 GC 策略（tracing GC）。
