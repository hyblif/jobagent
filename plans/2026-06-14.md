# 2026-06-14（日）Eval 集构建 + retrieval_eval.py

## 目标

拿到第一组可写进 README 的检索质量数字：加/不加 reranker 的 top-3 / top-5 命中率。

## 任务

- [ ] **eval 集构建**：`src/eval/eval_set.json`
  - 10-15 个固定面试问题，覆盖语料的不同主题。
  - 每题标注 gold：相关块的稳定标识（建议用 `源文件名 + heading` 而非 chroma id，
    重建索引后仍可比）。
  - 难度分布：直白题（问法和语料措辞接近）和换述题（同义改写、英文术语问中文语料）大约各半——换述题才能体现 reranker 价值。
- [ ] **实现 `src/eval/retrieval_eval.py`**
  - 对每题分别跑 `use_rerank=False`（纯向量 top-k）和 `use_rerank=True`。
  - 指标：hit@3、hit@5（gold 块出现在 top-k 即命中），顺手算 MRR。
  - 输出：终端表格（rich）+ `runs/eval/{date}.json` 落盘。
  - 入口：`uv run python -m src.eval.retrieval_eval`。
- [ ] 跑出第一组数字，记录在本文件底部。
- [ ] 单测：eval 指标计算函数（hit/MRR）的纯逻辑测试 2-3 个。

## 验收

- 一条命令出对比表：no-rerank vs rerank 的 hit@3 / hit@5 / MRR。
- 结果 JSON 落盘可复现。
- `uv run pytest` 全绿。

## 备注

- 如果 rerank 没有提升甚至下降：先检查换述题占比和 gold 标注，再检查 n_candidates
  是否太小（候选池都不含 gold 时 reranker 无能为力）。这本身就是 README 可写的分析。
